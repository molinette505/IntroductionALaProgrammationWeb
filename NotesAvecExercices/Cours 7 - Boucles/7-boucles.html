C'est quoi une boucle ?

Les boucles permettent de répéter un bloc de code plusieurs fois.

La boucle While (Tant que)

La boucle while est la plus simple

  • elle répète un bloc code tant que la condition entre parenthèses est vraie.
  • très utile lorsqu'on ne sait pas d'avance combien de fois on va boucler.
while (condition) {
    // code à répéter tant que la condition est vraie
}
  • La condition (condition):
    • Testée avant chaque tour de boucle.
    • Si elle est vraie, la boucle continue (on réexécute le code dans le corps de la boucle).
    • Si elle est fausse, la boucle s'arrête.
  • Le corps de la boucle { ... }: Le code à exécuter à chaque tour.
    • Il doit faire avancer la condition vers la fin (la condition doit pouvoir devenir false).
    • Sinon, la boucle pourrait tourner à l'infini !

Remarque : Le prochain exemple plus complexe démontre que la boucle while est idéale lorsque l'on ne sait pas à l'avance combien de tours seront nécessaires pour atteindre la condition recherchée.

Par exemple, dans un jeu de dés, on peut vouloir continuer à lancer le dé jusqu'à obtenir un 7. On ne sait pas à l'avance combien de tours seront nécessaires.

La boucle For (Pour)

Elle regroupe l'initialisation, la condition et l'incrémentation sur une seule ligne.

Utile lorsqu'on sait d'avance combien de fois on veut boucler (répéter).

for (initialisation; condition; incrémentation) { 
    // code à répéter tant que la condition est vraie
}
for (let i = 0; i <= 4; i++) { 
     // cette boucle s'exécutera de 0 à 4, soit 5 fois au total
}
  • L'initialisation :
    • À partir de quelle valeur on commence à compter?
    • On déclare et initialise une variable compteur à une valeur de départ. (ex: let i = 0)
    • On utiliser souvent i, j et k par convention
  • La condition :
    • Jusqu'à quelle valeur on boucle ?
    • Testée avant chaque tour de boucle. (ex: i < 5)
  • L'incrémentation :
    • De combien on augmente (ou diminue) la variable compteur à chaque tour ?
    • Modifie la variable de compteur à la fin de chaque tour. (ex: i++)
  • Le corps de la boucle { ... } :
    • Le code à exécuter à chaque tour.
Quel sera l'affichage dans la console ? 
for (let i = 1; i <= 4; i++) {
    console.log(i);
}
  • 1, 2, 3, 4
  • 0, 1, 2, 3, 4
  • 1, 2, 3

Parcourir un tableau avec une boucle for

On utilise la propriété .length pour dire à la boucle de s'arrêter pile à la fin du tableau.

On utilise la variable compteur i comme index pour accéder aux éléments du tableau.

Rappel sur les tableaux et les indices

  • L'index :
    • Chaque élément d'un tableau a un index qui indique sa position dans le tableau.
    • Il commence toujours à 0 pour le premier élément du tableau.
    • Le deuxième élément est à l'index 1, le troisième à l'index 2, etc.
  • Lire la valeur :
    • On accède à la valeur d'un élément du tableau en utilisant l'index entre crochets : tableau[index].
    • Par exemple, tableau[0] donne le premier élément, tableau[1] le deuxième, etc.
  • La taille d'un tableau :
    • La propriété .length donne le nombre d'éléments dans le tableau.
    • Si un tableau a 4 éléments, tableau.length vaut 4.
  • Le premier et dernier index :
    • Le premier index (position du premier élément) est toujours 0.
    • Le dernier index (position du dernier élément) est toujours tableau.length - 1.

Si ce n'est pas clair, revisiter les notes sur les tableaux.

Parcourir des tableaux parallèles

Astuce : Parcourir des tableaux parallèles

  • On parle de tableaux parallèles lorsqu'on utilise plusieurs tableaux de même longueur pour stocker des informations liées (ex : noms, notes, groupes).
  • Chaque indice i représente la même entité dans tous les tableaux : noms[i], notes[i], groupes[i] concernent le même étudiant.
  • Cette technique permet d'associer facilement des données sans objets, mais il faut toujours s'assurer que les tableaux ont la même longueur !
  • On utilise une seule boucle for pour parcourir tous les tableaux en même temps, en accédant à l'indice i dans chacun.

L'instruction Break

break sert à "casser" la boucle et à en sortir immédiatement, même si la condition de départ est encore vraie. Idéal pour optimiser une recherche.
