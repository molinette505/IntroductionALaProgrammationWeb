Introduction

Une fonction est un bloc de code qu’on peut réutiliser pour faire une tâche précise.

On l’écrit une fois, puis on l’appelle quand on en a besoin.

Pourquoi utiliser des fonctions ?

  • Éviter la répétition (principe DRY : Don’t Repeat Yourself).
  • Rendre le code plus clair (on lit des “actions” : calculerMoyenne()).
  • Faciliter la maintenance (si on corrige, on corrige à un seul endroit).

Image mentale : une petite “machine” dont l’intérieur est invisible

  • Elle reçoit des entrées (paramètres).
  • Elle fait un travail.
  • Elle peut renvoyer une sortie (avec return).

1) Déclarer et appeler une fonction

Il y a deux étapes : 1) déclarer la fonction (la créer), puis 2) l’appeler (l’exécuter).

Déclaration (création)

function nomFonction(param1, param2) {
  // instructions à exécuter
}
  • function : mot-clé pour créer une fonction.
  • nomFonction : le nom de la fonction (souvent un verbe).
  • param1, param2 : les entrées (facultatives).
  • Le code entre { ... } s’exécute à chaque appel.

Conseil : donne un nom qui décrit l’action : afficherScore, calculerTotal, verifierAcces.

Appel (exécution)

Pour exécuter la fonction : nomFonction() (le nom suivi de parenthèses).

2) Paramètres et arguments

Les paramètres servent à personnaliser une fonction.

On exécute la même fonction, mais avec des valeurs différentes.

Vocabulaire

  • Paramètre : variable écrite dans la définition function saluer(nom).
  • Argument : valeur donnée lors de l’appel saluer("Emma").

3) Retourner une valeur (return)

Une fonction peut retourner un résultat. Ce résultat peut ensuite être stocké dans une variable, affiché, ou réutilisé ailleurs.

À retenir

  • return renvoie une valeur.
  • Sans return, la fonction ne renvoie rien d’utile (souvent undefined).

Sortie immédiate (Early Exit)

Dès que JavaScript rencontre return, il quitte la fonction immédiatement. C’est utile pour arrêter vite quand une condition est remplie.

  • On peut avoir plusieurs return.
  • Le premier return exécuté arrête la fonction.

4) Portée des variables (scope)

À retenir

Une variable créée dans une fonction n’existe que dans cette fonction. On dit qu’elle est locale.

Important : la variable bonus n’existe pas en dehors de la fonction. C’est pour ça que la dernière ligne cause une erreur.

Mini récap

  • On déclare une fonction, puis on l’appelle.
  • Paramètres = entrées dans la définition, arguments = valeurs à l’appel.
  • return renvoie un résultat (et peut arrêter la fonction).
  • Une variable déclarée dans une fonction est locale (scope).
