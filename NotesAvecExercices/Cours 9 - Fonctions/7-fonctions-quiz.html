Pourquoi utiliser une fonction ?

  • Pour dupliquer le même bloc partout
  • Pour regrouper du code réutilisable
  • Pour éviter toute variable

Quelle syntaxe déclare correctement une fonction nommée ?

  • function saluer() {}
  • fonction saluer() {}
  • function = saluer() {}

Quelles affirmations sont vraies ?

  • Les paramètres sont définis dans la déclaration de la fonction.
  • Les arguments sont les valeurs passées à l'appel.
  • Une fonction ne peut recevoir qu'un seul argument.
  • Les paramètres existent uniquement dans la portée de la fonction.

Si function somme(a, b) { return a + b; }, que vaut somme(4, 6) ?

Une variable déclarée avec let dans une fonction est...

  • Globale partout
  • Locale à cette fonction
  • Accessible uniquement dans le HTML

Quelle écriture est une fonction fléchée valide ?

  • const doubler = (n) => n * 2;
  • const doubler => (n) n * 2;
  • arrow doubler(n) { return n * 2; }

Ordonne les étapes de conception.

  • Tester l'appel avec des arguments concrets
  • Identifier ce que la fonction doit recevoir
  • Écrire le corps qui calcule ou agit
  • Définir ce qu'elle doit retourner

Associe chaque terme à sa définition.

  • Nom de variable déclaré dans la signature
  • Valeur fournie lors de l'appel
  • Valeur renvoyée par la fonction
  • Zone où une variable est accessible
